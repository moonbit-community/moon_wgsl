## Description
Expression arena ordering is currently distributed across `wgsl_lower_function_expressions.mbt`, `wgsl_lower_calls.mbt`, and `wgsl_lower_materialization.mbt`. This caused repeated fixes for left-associative binary chains, function-argument-root projections, constants in call arguments, and short-circuit expression numbering.

## Design
Create an `ExpressionArenaScheduler` that takes semantic expression trees and produces deterministic Naga-compatible handle allocation order. It must be responsible for binary expression ordering, constructor argument ordering, projection root predeclaration, and constant/literal allocation order. Existing local predicate branches should be moved into scheduler tests or deleted when covered.

## Acceptance Criteria
- [ ] Binary expression ordering is decided by `ExpressionArenaScheduler`, not scattered helper predicates.
- [ ] Constructor, projection, and literal ordering use the same scheduler entry point.
- [ ] Existing representative traces for `fetch_point_shadow`, `fetch_spot_shadow`, `sample_directional_cascade`, `downsample_first`, and `rgb_to_hsv_trace` still pass.
- [ ] At least two old local ordering helper branches are deleted.

# ISS-012: Move statement emit ranges into BodyEmissionPlanner

## Metadata

- Status: closed

## Description
Statement emission ranges are currently inferred from lowered expression side effects and block-local watermarks. This caused parent block emit ranges to accidentally include expressions created inside nested control-flow blocks. The latest fix made max-expression calculation recursive, but the deeper problem is that body emission is still inferred after the fact.

## Design
Introduce `BodyEmissionPlanner` to compute statement order, emit ranges, call statements, stores, returns, control-flow child body ranges, and local declaration placement from `NagaCompatFunctionPlan`. Lowering should stop pushing emit ranges directly in unrelated materialization helpers.

## Acceptance Criteria
- [x] Body emission order is represented as structured plan data.
- [x] Nested `if`, `switch`, `loop`, and `block` expressions cannot be re-emitted by parent blocks.
- [x] Return-call argument preparation is planned in the correct block.
- [x] Existing body trace cases and external parity gates pass.

## Description
The workspace has a clear physical dependency direction, but a current-tree architecture audit found that compatibility concepts still leak into WGSL Core and that implementation seams are exposed as public interfaces. This contradicts ADRs 0002, 0003, 0009, and 0017 even though the existing architecture guardrails pass.

The leaks include naga-oil compose and preprocess contracts in `Milky2018/wgsl/common`, naga-oil directives and imports in the official WGSL AST, generated-import provenance and compose graph records in WGSL Semantic IR, internal Composer stages exposed through the facade, publishable implementation packages, and filesystem symlinks used as a writer ownership mechanism.

## Design
Restore the intended dependency and conceptual ownership:

- WGSL Core owns only official WGSL syntax, semantic validation, semantic IR, and neutral writer machinery.
- The Naga Compatibility Layer owns all Naga-shaped provenance, arena ordering, naming, and writer context.
- The naga-oil Compatibility Layer owns preprocessing syntax, imports, composition contracts, symbol graphs, final names, source edit implementation, and profile defaults.
- The Moon WGSL Facade owns a small user-facing interface instead of re-exporting an implementation type.
- Internal implementation seams remain internal and are tested through the owning module's external interface.

Prefer a documented synchronized breaking release over compatibility aliases that preserve the leaks.

## Acceptance Criteria

- [x] The facade exposes only user workflows and no internal pipeline, trace, writer-plan, or source-edit seams.

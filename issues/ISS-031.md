## Description
The naga-oil composer is symbol-binding-first, but reference liveness still has source-assisted semantic scans. `WgslComposeSemanticFacts` and import request planning still consult source strings for unqualified reference names, reference paths, and colon-pair context.

These scans are not the old raw text replacement path, but they still mean the Symbol Graph is not the only source of semantic truth.

## Design
Make parsed and bound reference facts the sole input for alias/import/name reachability. Source text may be preserved for diagnostics and final source-map material, but compose liveness, import retention, alias redirects, and qualified reference decisions must use structured facts derived from the parser/AST and symbol binding.

## Acceptance Criteria
- [ ] `wgsl_semantic_source_contains_unqualified_reference_name`, `wgsl_semantic_source_contains_reference_path`, and colon-pair source helpers are removed from compose liveness.
- [ ] `active_wgsl_full_module_import_dependency_scan_source` is replaced by structured bound reference facts.
- [ ] Import request filtering reads `WgslComposeSemanticFacts` or Symbol Graph data, not raw source snippets.
- [ ] Source Edit Backend consumes an already-bound `EmissionPlan` and does not decide semantic reachability.
- [ ] External naga-oil compose parity passes with `cases=170` or a documented higher current count, and no new source-level liveness fallback is introduced.

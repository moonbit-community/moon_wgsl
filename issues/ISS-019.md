## Description
Convert the repository from a single publishable MoonBit module into a non-publishable workspace root with publishable member modules under `modules/`.

## Design
Create the physical workspace structure first, in one atomic migration commit. The root owns workspace docs, issues, testdata, tools, and CI. Publishable modules live under `modules/wgsl`, `modules/moon_wgsl_naga`, `modules/moon_wgsl_naga_oil`, and `modules/moon_wgsl`.

## Acceptance Criteria
- [x] A workspace manifest exists at the repository root and includes all publishable member modules.
- [x] The repository root is not the `Milky2018/moon_wgsl` publishable module.
- [x] `modules/wgsl`, `modules/moon_wgsl_naga`, `modules/moon_wgsl_naga_oil`, and `modules/moon_wgsl` each have a valid `moon.mod`.
- [x] Large corpora, oracle tools, issues, and docs remain at workspace root.
- [x] `moon info --target all` can run from the workspace root without module-boundary confusion.

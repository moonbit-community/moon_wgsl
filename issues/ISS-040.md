## Description
`directive`, `import_syntax`, `transform`, `source_rewrite`, and `import_substitution` publish implementation method and record inventories even though regular callers do not use them. They are in-process implementation seams of preprocess and compose, not external interfaces.

## Design
Move these packages under an internal namespace or merge them into their owning deep modules. Keep implementation seams private. Migrate tests to preprocess/compose observable behavior, retaining white-box tests only inside the owning package when they protect a complex invariant.

The public naga-oil package inventory should contain only user contracts, preprocess, metadata, resolver/registry where independently useful, compose, export, and the explicit diagnostics adapter.

## Acceptance Criteria
- [x] Directive parsing, import syntax, transform, source rewrite, and import substitution are not publishable public interfaces.
- [x] Their public declaration inventories are removed from the architecture's allowed external packages.
- [x] External module tests exercise preprocess/compose results instead of constructing internal state.
- [x] No facade or tool imports an internal implementation package.
- [x] Source Edit Backend remains implementation-only and consumes an already-bound emission plan.
- [x] naga-oil checks, workspace tests, architecture guardrails, and strict external parity pass.

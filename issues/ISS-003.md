# ISS-003: Trace function body execution plans against upstream

## Description
Representative trace currently compares expression order, argument bindings, and materialized expressions for selected functions. It does not fully prove statement emission order, local declaration placement, control-flow block construction, helper local policy, or source-span-independent materialization decisions.

## Design
Extend the writer trace to emit a structured body plan for each function. The plan should include statement index, statement kind, expression range, materialized result handle, local declaration handle, temp name, source-local or generated-local provenance, and whether the statement is emitted, skipped, or folded.

The trace must be compared against upstream oracle for representative functions before byte drift fixes are attempted.

## Acceptance Criteria
- [ ] The trace gate records statement plan rows for upstream oracle and Moon.
- [ ] Materialization decisions include the reason and input expression handle.
- [ ] Local declaration order and helper temp order are trace-compared, not inferred from output text.
- [ ] Existing representative trace cases continue to pass under the stronger trace.

# ISS-013: Make the Naga-compatible writer consume plans only

## Metadata
- Type: task
- Status: closed
- Priority: 1

- Assignee: unassigned

- Updated: 2026-06-05

## Description
The writer still interprets lowered IR and body-plan side effects to decide temporary names, local declarations, and emission order. This keeps writer byte parity coupled to lowerer accidents.

## Design
Refactor the Naga-compatible writer to consume `NagaCompatFunctionPlan`, `BodyEmissionPlanner`, and final name data only. The writer should not rediscover materialized expressions, local order, or emit ranges from raw IR. The generic writer may have its own adapter, but it must not mutate or reinterpret the compatibility plan.

## Acceptance Criteria
- [x] Naga-compatible writer has a plan-only entry path for function bodies.
- [x] Temporary name allocation comes from the plan.
- [x] Local declaration order comes from the plan.
- [x] Writer output does not depend on scanning raw lowerer side effects.
- [x] Representative trace and external parity gates pass.

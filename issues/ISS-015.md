## Description
The strict external naga-oil compose gate has reached zero writer/order/name drift, but full byte output still has known drift. Drift manifests are no longer acceptable as compatibility proof. The remaining byte drift must be reduced to zero against the pinned upstream oracle, then the legacy known-drift mode and manifests must be removed.

This issue is migrated from the legacy tracker row `WGSL-281`.

## Design
Treat byte drift as a symptom of upstream execution-model mismatch, not as formatting inventory. Use the representative trace, module arena trace, body plan trace, and byte-fragment trace to identify shared root causes. Fix shared model gaps before refreshing manifests, and refresh manifests only from observed gate output while the legacy known-drift mode still exists.

Non-goal: adding new allowlisted drift rows as a steady-state compatibility story.

## Acceptance Criteria
- [x] `bash tools/check_external_naga_oil_compose_parity.sh` passes in strict mode without `MOON_WGSL_ALLOW_KNOWN_DRIFT=1`.
- [x] `testdata/external_naga_oil_compose_byte_drift.tsv` and related legacy known-drift mode are removed or made unreachable from default CI.
- [x] `bash tools/check_external_naga_oil_drift_taxonomy.sh` no longer tracks accepted byte drift inventory.
- [x] The final fix path is explained through trace/model changes, not per-file output patches.

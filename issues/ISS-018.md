## Description
The current external corpus gate has no known skips or expected failures for its tracked five repositories, but that does not exhaust real-world shader construction patterns. The legacy tracker kept `WGSL-254` open to continue expanding real-project WGSL corpora and profile shapes beyond the current gate.

This issue is migrated from the legacy tracker row `WGSL-254`.

## Design
New coverage must be manifest-owned and exact-gated. Preserve `skipped=0`, `expected-failures=0`, manifest-owned expected-invalid classifications, manifest-owned raw-vs-compose profile execution modes, exact per-repository corpus counts, and strict TSV schema validation. Add new corpora only with concrete materialization/profile rules that match how those projects actually construct shader modules.

## Acceptance Criteria
- [ ] At least one additional real-project corpus or profile family is added through exact manifests.
- [ ] New corpus/profile entries preserve `skipped=0` and `expected-failures=0`.
- [ ] Raw-vs-compose execution mode is explicit for every new profile.
- [ ] Expected-invalid and oracle-blocked classifications remain manifest-owned and schema-checked.
- [ ] External corpus gates pass with exact per-repository counts.

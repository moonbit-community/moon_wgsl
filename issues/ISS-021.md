## Description
Replace the current mixed IR model with separate WGSL Semantic IR and Naga Compatibility View models.

## Design
WGSL Semantic IR represents program meaning and belongs to `Milky2018/wgsl`. Naga compatibility owns Naga-shaped arena/order/temp-name/writer plans and belongs to `Milky2018/moon_wgsl_naga`. Compatibility provenance, import arena events, writer traces, and byte-parity policy must not remain flags inside the core IR.

## Acceptance Criteria
- [x] WGSL Semantic IR contains no Naga writer policy, naga-oil writer policy, or byte-parity trace state.
- [x] Naga Compatibility View is derived from WGSL Semantic IR through an explicit conversion boundary.
- [x] Naga arena ordering, temporary names, and writer plans live outside `Milky2018/wgsl`.
- [x] Existing IR roundtrip and validation tests are reassigned to the owning module.
- [x] Boundary tests prove `Milky2018/wgsl` cannot import Naga or naga-oil compatibility modules.

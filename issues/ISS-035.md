## Description
The current ADRs state the intended architecture, but the architecture metadata only checks package dependency arrows and selected names. It does not detect compatibility concepts owned by WGSL Core, methods leaked through a re-exported type, or internal packages accidentally published as interfaces.

Define the final interfaces and turn conceptual ownership into machine-checked negative constraints before moving code.

## Design
Add one ADR that supersedes the weak interpretation of ADRs 0009, 0017, and ISS-030. Extend the structured architecture manifest and checker with explicit allowed public packages, forbidden public concepts, facade method inventories, and writer source ownership.

The guard must initially describe the target architecture while allowing named migration exceptions. Each subsequent issue removes its own exception. The final state must have no compatibility exception under WGSL Core.

## Acceptance Criteria
- [x] A target architecture ADR assigns every leaked concept and public workflow to one owning module.
- [x] The ADR specifies the final Naga compatibility context and facade interfaces, including invariants and error ownership.
- [x] Structured architecture metadata can inspect complete exported method sets, not only direct facade declarations.
- [x] The manifest records allowed public packages and forbidden concepts for WGSL Core, Naga, naga-oil, and facade modules.
- [x] Temporary migration exceptions are explicit, issue-linked, and rejected when their issue closes.
- [x] The updated architecture checker and tracker freshness gate pass.

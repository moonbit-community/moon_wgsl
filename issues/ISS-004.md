## Description
The external compose gate reports `writer-drift=0` but `byte-drift=55`. That means symbol/order/name level checks are passing while byte-level output still differs. The remaining drift cannot be responsibly fixed by output string patches; the writer needs byte-fragment trace evidence that explains exactly which policy diverges.

## Design
Add a byte writer trace that records final fragments with their owning declaration, statement, expression, token policy, whitespace policy, literal spelling policy, and generated-name source. Compare upstream and Moon fragment streams before diffing final text.

This should be a separate trace layer from expression arena traces so it can identify whitespace/token/literal differences after semantic execution already matches.

## Acceptance Criteria
- [x] The byte writer can emit a machine-readable fragment stream for both upstream oracle and Moon.
- [x] Fragment rows include semantic owner identity and token policy source.
- [x] The gate reports first divergent fragment with surrounding semantic context.
- [x] At least five current byte drift files are classified by fragment trace root cause.

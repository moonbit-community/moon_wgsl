## Description
The workspace split introduced the right module boundaries, but several writer implementation files are still copied between `modules/wgsl/ir` and `modules/moon_wgsl_naga`. Identical or near-identical writer substrate makes the Generic WGSL Writer and Naga Compatibility Layer vulnerable to silent divergence.

The public boundary should remain split, but shared writer mechanics should have one source of truth.

## Design
Move neutral writer substrate into WGSL Core or a private shared package that both writer frontends consume. Keep Naga-specific ordering, naming, and byte-parity policy inside `Milky2018/moon_wgsl_naga`. The generic writer should not carry Naga policy, and the Naga writer should not copy generic formatting utilities just to avoid a package dependency.

## Acceptance Criteria
- [x] Identical writer files such as derived-view, filter, expression-type, and type emission helpers have one implementation source.
- [x] `Milky2018/wgsl` owns only neutral writer primitives and does not expose Naga compatibility policy.
- [x] `Milky2018/moon_wgsl_naga` owns Naga compatibility ordering, naming, materialization, and byte-parity policy.
- [x] A guard or test prevents future silent copy/paste drift between generic and Naga writer substrate.
- [x] `bash tools/check_naga_writer_representative_trace.sh` and workspace MoonBit checks pass.

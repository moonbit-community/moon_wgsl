## Description
Whitebox tests have grown with each drift class. They are useful as regressions but they mostly assert symptoms of current cases. This encourages adding one more test and one more helper branch instead of proving a reusable model.

## Design
Add model invariant gates for arena scheduling, call argument planning, short-circuit planning, body emission planning, and writer naming. Existing representative traces should stay, but repetitive local assertions should be replaced with table-driven fixtures that validate the plan output.

## Acceptance Criteria
- [ ] Plan dump tests exist for arena, calls, short-circuit, body emission, and names.
- [ ] At least one cluster of repetitive wbtests is replaced by table-driven plan fixtures.
- [ ] Representative trace manifest remains the end-to-end parity gate.
- [ ] Test coverage no longer requires adding a bespoke wbtest for every drift class.

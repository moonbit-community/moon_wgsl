## Description
The official WGSL AST and parser currently model `ifdef`, `ifndef`, `define`, `define_import_path`, and `import` as WGSL directives and declarations. These are naga-oil preprocessing extensions, not official WGSL language syntax, so compatibility dialect changes alter the WGSL Core interface.

## Design
Make WGSL Core parse official directives only: `enable`, `requires`, and `diagnostic`. Move extension-line recognition, import grammar, conditional preprocessing syntax, and mixed-source preprocessing nodes into a naga-oil-owned frontend.

The naga-oil frontend must preprocess or compose extension-bearing source into official WGSL before invoking WGSL Core. Reuse the neutral WGSL lexer only where it does not require extension variants in the official AST. Preserve source spans and diagnostics in the naga-oil layer.

## Acceptance Criteria
- [x] Official WGSL directive and global-declaration enums contain no naga-oil variants.
- [x] `wgsl/directive_syntax` and `wgsl/import_syntax` are removed, renamed to official-only functionality, or moved under `moon_wgsl_naga_oil`.
- [x] WGSL Core rejects or reports extension directives as non-official syntax without interpreting them.
- [x] naga-oil preprocess and compose workflows continue to support all current extension syntax with source spans and diagnostic parity.
- [x] Core parser tests cover only official WGSL; dialect tests live in naga-oil packages.
- [x] CTS/Naga parser gates and strict external naga-oil parity pass.

# ISS-008: Introduce NagaCompatFunctionPlan as the sole function lowering contract

## Description
Function lowering currently appends expressions, materializes temporaries, and pushes statement emit ranges as interleaved side effects. This makes it possible to fix every drift with another local predicate. A future agent needs a single contract that describes the whole Naga-compatible function body before writer emission.

## Design
Add `NagaCompatFunctionPlan` as the structured output of function lowering. The plan should contain expression arena allocation, call argument schedule, short-circuit locals, materialized expressions, body emission order, local declaration order, and generated name slots. Existing lowerer paths may populate the plan during migration, but downstream writer logic must move toward consuming the plan.

## Acceptance Criteria
- [ ] `NagaCompatFunctionPlan` exists as a concrete type with explicit fields for arena, call, short-circuit, materialization, body, and name planning.
- [ ] At least one representative function is emitted through the plan without using ad hoc lowerer state.
- [ ] The plan can be dumped in tests for stable golden comparison.
- [ ] No public API promises are made around the transitional internal shape.

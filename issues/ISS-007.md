## Description
The current Naga parity work is still vulnerable to rule-patch growth. Expression lowering, call argument ordering, short-circuit materialization, body emission ranges, and writer temporary naming are spread across multiple large modules. Each newly discovered upstream drift can still be fixed by adding another local predicate, which grows code without turning upstream behavior into a single executable model.

This epic tracks a destructive refactor that replaces scattered parity rules with one structured Naga compatibility execution model.

## Design
Build a single `NagaCompatFunctionPlan` pipeline. Lowering should first produce structured plans for arena allocation, call arguments, short-circuit values, materialization, body emission, and names. The writer must consume those plans instead of reinterpreting historical lowerer side effects. New parity fixes must modify the model, not add local rule predicates.

Non-goal: preserving compatibility with the current internal lowerer shape. Public API compatibility is not required for this refactor.

## Acceptance Criteria
- [ ] No new local rule-patch predicates are required to fix representative Naga drift cases.
- [ ] Expression arena order, call argument order, short-circuit materialization, body emission, and temp naming are represented in structured plans.
- [ ] Existing focused drift cases pass through the unified model.
- [ ] Obsolete helper branches in the old lowerer/materializer are deleted or clearly isolated behind the new plan.
- [ ] `moon test`, representative trace, external parity, and taxonomy gates pass.

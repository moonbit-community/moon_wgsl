## Description
The current repository cannot honestly claim that the Moon WGSL execution model is identical to upstream Naga or naga-oil. Recent validation proves important slices: representative writer trace parity passes for 20 cases, external naga-oil compose parity has `writer-drift=0`, and the Bevy corpus composes without current validation failures. However, strict byte parity still reports `byte-drift=55`, and existing trace gates compare only selected expression order, bindings, and materialized temporaries.

This epic tracks the work required to turn execution-model parity from an assumption into a checked property.

## Design
Do not solve this by editing individual expected strings or case-specific writer policy. Build upstream-comparable execution traces for each stage, then align the internal model where traces diverge. The tracked stages are module/declaration arena construction, function body lowering and statement plan, materialization and temp-name planning, and final byte writer fragments.

## Acceptance Criteria
- [ ] The tracker contains one issue per unproven execution stage.
- [ ] A future agent can run gates that compare upstream and Moon execution traces, not only final text.
- [ ] Remaining byte drift cases are classified by trace-derived root cause, not by filename.
- [ ] The project can state exactly which execution stages are proven equivalent and which are not.

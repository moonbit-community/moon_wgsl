## Description
Add a top-level `publish.mbtx` script that updates and publishes all publishable workspace modules with the same version.

## Design
The script accepts a version, updates each module `moon.mod`, runs formatting/info/check validation, and publishes in dependency order: `wgsl`, `moon_wgsl_naga`, `moon_wgsl_naga_oil`, then `moon_wgsl`. Failure stops the script and reports the last completed module without trying to roll back external publish state.

## Acceptance Criteria
- [ ] `publish.mbtx` validates version input.
- [ ] `publish.mbtx` updates all publishable member `moon.mod` files to the same version.
- [ ] `publish.mbtx` runs required pre-publish validation.
- [ ] `publish.mbtx` publishes in dependency order.
- [ ] Partial publish failures are reported clearly without automatic rollback.

## Close Notes
Closed on 2026-06-12. Added top-level `publish.mbtx`, runnable as `moon run --target js publish.mbtx <version>`. It validates semver input, updates the four publishable module manifests and their internal dependency versions, runs workspace `fmt`, `info`, `check`, and `test`, then publishes `wgsl`, `moon_wgsl_naga`, `moon_wgsl_naga_oil`, and `moon_wgsl` in dependency order. Verification: `moon run --target js publish.mbtx` compiled and produced the expected usage error without mutating manifests.

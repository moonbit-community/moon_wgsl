## Description
ISS-030 removed source copies by making files under `moon_wgsl_naga` symlink to WGSL Core. This prevents drift but creates filesystem-level ownership leakage: editing a Naga path mutates Core implementation, package locality is false, and the architecture manifest preserves the coupling as an invariant.

## Design
Give neutral writer mechanics one explicit code owner. Prefer a precisely named neutral package inside WGSL Core that exports only the minimum implementation interface required by the Generic WGSL Writer and Naga adapter. Naga-specific ordering, naming, compatibility views, and byte policy remain in `moon_wgsl_naga`.

Do not replace symlinks with copies. If the neutral interface would expose most of the implementation, merge the generic writer substrate into the owning Core package and adapt Naga at a smaller derived-view seam.

## Acceptance Criteria
- [x] No source symlink is used to share writer implementation.
- [x] Neutral writer mechanics have one explicit package and repository path owner.
- [x] The interface consumed by the Naga adapter contains no Naga policy and is smaller than the shared implementation.
- [x] Generic and Naga writer tests exercise their respective external interfaces.
- [x] Architecture metadata checks ownership and rejects symlinks or duplicate substrate files.
- [x] Generic writer, Naga trace, workspace tests, and strict external parity pass.

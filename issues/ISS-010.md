# ISS-010: Replace call argument predicates with CallArgumentPlanner

## Metadata

## Description
Call argument lowering currently relies on pre-pass and lookahead predicates. Recent fixes had to distinguish user-call value arguments, direct function arguments, inline named constants, literals, alias projections, and reference arguments. This structure is still vulnerable to more rule patches.

## Design
Introduce `CallArgumentPlanner`. For every call, it should produce a complete argument schedule: allocation order, materialization decision, inline decision, pointer argument handling, and argument-to-parameter typing. `lower_user_function_call_arguments` should delegate ordering to the planner instead of running local scans.

## Acceptance Criteria
- [x] Call argument order is represented as a structured schedule.
- [x] Pointer arguments, direct function arguments, inline named constants, literals, alias projections, and reference arguments are modeled explicitly.
- [x] Existing `fetch_spot_shadow`, `sample_directional_cascade`, `downsample_first`, `downsample_second`, and `indexed-reference-call-args` trace cases pass.
- [x] Local lookahead predicates in `wgsl_lower_calls.mbt` are deleted or reduced to planner internals.

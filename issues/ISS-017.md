## Description
Successful compose/preprocess output parity does not prove failure behavior parity. The legacy tracker recorded that compose/preprocess errors are not byte-gated against the pinned `naga_oil` oracle across the external compose corpus, so diagnostic category, message, source location, or byte formatting can drift unnoticed.

This issue is migrated from the legacy tracker row `WGSL-284`.

## Design
Use `bevyengine/naga_oil@bc444c82bb593ede94c55cdbf799e9743800843e` as the sole oracle for negative cases. Add manifest-owned negative compose/preprocess fixtures and compare machine-readable diagnostics plus final diagnostic bytes. The gate should reject unchecked diagnostic drift by default.

## Acceptance Criteria
- [ ] Negative compose/preprocess cases are manifest-owned and schema-checked.
- [ ] Moon and oracle diagnostics are compared for category, message, source span/location, and rendered bytes.
- [ ] Diagnostic drift fails CI by default, with no passive allowlist mode for unchecked failures.
- [ ] Existing success-path parity gates remain independent from diagnostic parity gates.

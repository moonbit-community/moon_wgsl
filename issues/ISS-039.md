## Description
The facade directly re-exports naga-oil's `Composer` type. MoonBit therefore exposes the type's full public method set, including before-IR stages, parity writers, and trace plans used by tools and white-box tests. The facade is syntactically thin but conceptually shallow.

## Design
Define a facade-owned user module with a small interface for source registration, preparation, composition, and export. It delegates to a private naga-oil implementation and maps lower-level errors into facade-owned errors.

Move tracing, before-IR inspection, and parity execution to a dedicated diagnostics adapter outside the facade. Tools may import that diagnostics package directly; normal callers and facade tests must use only observable user workflows.

## Acceptance Criteria
- [x] The facade no longer re-exports the naga-oil `Composer` implementation type.
- [x] The facade-owned interface has only user workflow entry points and documented ordering/error invariants.
- [x] Trace, parity, before-IR, writer-plan, graph, and source-edit methods are absent from the facade's complete method inventory.
- [x] Tools use a dedicated diagnostics adapter rather than methods on the user-facing object.
- [x] Tests at the facade seam replace tests that depend on internal stages.
- [x] Facade checks, workspace tests, architecture guardrails, and strict parity pass.

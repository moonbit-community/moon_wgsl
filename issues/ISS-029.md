## Description
The WGSL lowerer still concentrates too many semantics in a small set of very large files. `modules/wgsl/ir/wgsl_lower_function_expressions.mbt` and `modules/wgsl/ir/wgsl_lower_statements.mbt` own expression lowering, constant folding, call argument planning, materialization, statement emission order, and several trace-derived Naga compatibility ordering fixes.

This makes future parser, IR, and writer changes risky because local expression fixes can silently change arena order, temporary names, or statement output far away from the modified branch.

## Design
Replace the overloaded lowerer shape with explicit planner layers that expose small contracts:

- module and global predeclaration planning
- type lowering
- function signature and body lowering
- statement planning
- expression arena scheduling
- const evaluation
- call and builtin planning
- side-effect materialization
- temporary naming provenance

Trace-derived Naga compatibility behavior must be represented as planner invariants or deleted by a unified model, not embedded as long case-specific helper names inside expression lowering.

## Acceptance Criteria
- [ ] `wgsl_lower_function_expressions.mbt` is split so expression lowering, expression arena scheduling, call planning, const evaluation, and materialization each have one clear owning package/file.
- [ ] `wgsl_lower_statements.mbt` is split so control-flow lowering and body emission planning are separate responsibilities.
- [ ] Trace-derived helpers such as `lower_function_binary_const_parent_between_right_const_and_local_reference` are removed or replaced by named planner rules with tests that describe the invariant rather than the fixture.
- [ ] Statement emission uses explicit body plans and no longer asks expression lowering code to make final writer-order decisions.
- [ ] `moon check --target all --deny-warn`, `moon test --target all`, `bash tools/check_naga_writer_representative_trace.sh`, and `bash tools/check_external_naga_oil_compose_parity.sh` pass.

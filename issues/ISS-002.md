## Description
The current Naga-compatible writer builds a `WgslIrNagaWriterModule`, but there is no gate proving its module-level execution order matches upstream. Declaration import/localization order, predeclared type and constant slots, global variables, function handles, and entry-point ordering may still differ even when selected function expression traces match.

## Design
Add an upstream-comparable module arena trace. It should be generated for the pinned oracle and for Moon from the same compose input. The trace must include declaration kind, original provenance, generated name, final handle/index, root inclusion decision, and dependency edges used to schedule declaration emission.

Do not infer provenance from final generated names. Use structured symbol identity and module-relative source paths.

## Acceptance Criteria
- [x] A machine-readable module arena trace exists for upstream oracle and Moon.
- [x] The trace covers type, constant, override, global variable, function, entry point, and const-assert arena slots.
- [x] At least one complex Bevy PBR compose case is added to the module-arena trace gate.
- [x] Divergence output identifies the first differing arena event with provenance and dependency context.

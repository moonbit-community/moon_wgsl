# ISS-038: Build the Naga compatibility view outside semantic IR

## Description
WGSL Semantic IR currently stores generated-import provenance and import-arena events, and the WGSL Core package owns final-name and link-graph records used by naga-oil. The Core lowerer accepts compatibility metadata and changes declaration flags and ordering based on it. This directly violates ADR 0009.

## Design
Make `parse_wgsl_module_to_ir` the sole semantic lowering entry point and remove compatibility inputs and fields from `@ir.Module`. The Naga module owns immutable compatibility input records and derives a `NagaCompatibilityView` from semantic IR plus that context.

The naga-oil composer owns its Compose Graph, Symbol Graph, Final Name Table, import events, and source provenance. It maps only the Naga-relevant facts into the Naga-owned context. Generic WGSL lowering and writing must not observe those facts.

If compatibility requires declaration ordering different from semantic source order, represent that entirely in the Naga view rather than mutating or annotating semantic IR.

## Acceptance Criteria
- [x] `parse_wgsl_module_to_ir` is the only public semantic lowering entry point.
- [x] `@ir.Module` and its declarations contain no generated-import, compose-graph, Naga-name, or import-arena fields.
- [x] Final-name, link-graph, import-event, and generated-provenance types are owned by Naga or naga-oil according to their semantics.
- [x] The Naga interface accepts semantic IR plus an opaque Naga-owned compatibility context and constructs the compatibility view internally.
- [x] Generic WGSL roundtrip behavior is independent of compatibility context.
- [x] Semantic IR tests, Naga trace tests, architecture guardrails, and strict byte parity pass.
